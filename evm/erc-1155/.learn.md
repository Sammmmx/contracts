# ERC-1155 Multi-Token Contract

## What is ERC-1155?

ERC-1155 is a multi-token standard that allows a single contract to manage multiple token types simultaneously — both fungible and non-fungible — under one address.

Before ERC-1155, you needed a separate contract for every token type. A game with swords, shields, and gold coins required three deployments. ERC-1155 collapses all of that into one contract, identified by token IDs.

- Token ID `1` → Gold Coins (fungible, supply of 1,000,000)
- Token ID `2` → Silver Sword (supply of 500)
- Token ID `3` → Legendary Dragon (supply of 1, effectively an NFT)

## Core Mechanisms

### Role-Based Access Control

Instead of a single owner, this contract uses three distinct roles via OpenZeppelin's `AccessControl`:

| Role                 | Purpose                            |
| -------------------- | ---------------------------------- |
| `DEFAULT_ADMIN_ROLE` | Grants and revokes all other roles |
| `MINTER_ROLE`        | Can call `mint` and `mintBatch`    |
| `URI_SETTER_ROLE`    | Can call `setURI`                  |

Roles are defined as `bytes32` constants derived from `keccak256` hashes:

```solidity
bytes32 public constant MINTER_ROLE = keccak256("MINTER_ROLE");
```

### Token URI

ERC-1155 uses a single base URI for all token types. Clients automatically substitute `{id}` with the token ID when fetching metadata:

```
ipfs://QmYourHash/{id}.json
```

The `EmptyURI` custom error prevents accidentally wiping metadata with an empty string.

### Batch Minting

`mintBatch` mints multiple token IDs in a single transaction. More gas efficient than calling `mint` repeatedly. `ids` and `amounts` must be equal length arrays — reverts with `ERC1155InvalidArrayLength` otherwise.

### supportsInterface Override

Both `ERC1155` and `AccessControl` implement `supportsInterface`. Solidity requires you to explicitly resolve this conflict:

```solidity
function supportsInterface(bytes4 interfaceId)
    public view override(ERC1155, AccessControl) returns (bool)
{
    return super.supportsInterface(interfaceId);
}
```

## Security Considerations

**Admin Role is Powerful**
The `DEFAULT_ADMIN_ROLE` can grant and revoke any role. In production, assign this to a MultiSig rather than an EOA.

**Role Separation**
A compromised minter cannot change the URI. A compromised URI setter cannot mint. Keep roles on separate addresses.

**No Supply Cap**
This contract does not enforce a maximum supply per token ID. Add a `mapping(uint256 => uint256)` supply tracker inside `mint` and `mintBatch` if needed.

**No Transfer Restrictions**
Tokens are freely transferable by default. Override `safeTransferFrom` and `safeBatchTransferFrom` if you need soulbound or restricted tokens.

**Receiver Hooks**
ERC-1155 calls `onERC1155Received` on contract recipients. Tokens cannot be sent to contracts that don't implement this hook — intentional protection against locked tokens.

## Deployment

```bash
npx hardhat ignition deploy ignition/modules/ERC1155Deploy.ts --network sepolia
```

With parameters:

```bash
npx hardhat ignition deploy ignition/modules/ERC1155Deploy.ts --network sepolia \
  --parameters '{"defaultAdmin":"ADMIN_ADDRESS","minter":"MINTER_ADDRESS","uriSetter":"URI_SETTER_ADDRESS"}'
```

Verify on Etherscan:

```bash
npx hardhat verify --network sepolia <DEPLOYED_ADDRESS> <ADMIN> <MINTER> <URI_SETTER>
```

## Testing Coverage

```bash
npx hardhat test
```

**Deployment** — Verifies correct role assignment per address at construction and empty initial URI.

**setURI** — Happy path, multiple updates, empty string revert with `EmptyURI`, unauthorized caller reverts with `AccessControlUnauthorizedAccount`.

**mint** — Single mints, balance accumulation, multiple token IDs, zero amounts, `MaxUint256` boundary values, zero address revert, unauthorized caller reverts.

**mintBatch** — Batch minting, per-ID balance verification, mismatched array length revert with `ERC1155InvalidArrayLength`, empty arrays, unauthorized caller reverts.

**AccessControl** — Full role lifecycle: grant, revoke, renounce. Newly granted addresses gain access, revoked addresses lose access. `RoleGranted` and `RoleRevoked` events validated. Non-admin role management reverts.

**supportsInterface** — ERC-1155, MetadataURI, AccessControl, and ERC-165 return true. ERC-721 and dummy interface return false.
